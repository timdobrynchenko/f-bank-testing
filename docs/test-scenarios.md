# Тестовые сценарии F-Bank

5 детальных сценариев: 3 на найденные дефекты + 2 на ключевые проходящие пути для покрытия.

| ID | Название | Приоритет | Назначение |
|---|---|---|---|
| [TS-01](#ts-01) | Расчет комиссии для суммы 91 рублей: некорректное округление | 🟡 Major | Баг #1 (округление комиссии) |
| [TS-02](#ts-02) | Попытка перевода отрицательной суммы | 🔴 Critical | Баг #2 (отрицательная сумма) |
| [TS-03](#ts-03) | Ввод 17 цифр в поле номера карты | 🟡 Major | Баг #3 (длина номера карты) |
| [TS-04](#ts-04) | Успешный перевод валидной суммы | 🔴 Critical | Happy path (успешный перевод) |
| [TS-05](#ts-05) | Сообщение об ошибке при превышении доступной суммы | 🟡 Major | Sad path (превышение доступной суммы) |

## TS-01
**Расчет комиссии для суммы 91 рублей: некорректное округление**

**Приоритет**: 🟡 Major

**Описание**: Проверка корректности расчета комиссии 10% с округлением вниз для суммы перевода, не кратной 10. Согласно спецификации, комиссия = floor(сумма * 0.1).

**Предусловия**:
- Сервис F-Bank запущен на http://localhost:8000
- Используется браузер Google Chrome (последняя версия).

| # | Шаг | Ожидаемый результат |
|---|---|---|
| 1 | Открыть URL http://localhost:8000/?balance=100&reserved=0  | Загружается страница F-Bank с тремя карточками валют. На рублевой карточке: «На счету: 100 руб., Резерв: 0 руб.» |
| 2 | Кликнуть на карточку «Рубли» | Справа открывается форма «Перевод на карту» с полем «Номер карты». |
| 3 | В поле «Номер карты» ввести 16 цифр (например, 1234567890123456) | Под полем «Номер карты» появляется поле «Сумма перевода». |
| 4 | В поле «Сумма перевода» ввести значение 91 | Под полем суммы отображается строка «Комиссия: 9 руб.» (floor(91 * 0.1) = floor(9.1) = 9). Появляется кнопка «Перевести». |

## TS-02
**Попытка перевода отрицательной суммы**

**Приоритет**: 🔴 Critical

**Описание**: Проверка валидации поля суммы перевода: ввод отрицательного значения должен быть заблокирован, поскольку перевод на отрицательную сумму нарушает бизнес-логику банковского сервиса.

**Предусловия**:
- Сервис F-Bank запущен на http://localhost:8000.
- Используется браузер Google Chrome (последняя версия).

| # | Шаг | Ожидаемый результат |
|---|---|---|
| 1 | Открыть URL http://localhost:8000/?balance=100&reserved=0  | Загружается страница F-Bank. На рублевой карточке: «На счету: 100 руб., Резерв: 0 руб.» |
| 2 | Кликнуть на карточку «Рубли» | Открывается форма «Перевод на карту». |
| 3 | В поле «Номер карты» ввести 16 цифр (например, 1234567890123456) | Появляется поле «Сумма перевода». |
| 4 | В поле «Сумма перевода» ввести значение -100 | Поле блокирует ввод знака «минус» / отображает сообщение об ошибке валидации. Кнопка «Перевести» не отображается. |

## TS-03
**Ввод 17 цифр в поле номера карты**

**Приоритет**: 🟡 Major

**Описание**: Проверка валидации длины номера карты. Согласно спецификации, поле должно принимать ровно 16 цифр; ввод дополнительных цифр должен блокироваться.

**Предусловия**:
- Сервис F-Bank запущен на http://localhost:8000.
- Используется браузер Google Chrome (последняя версия).

| # | Шаг | Ожидаемый результат |
|---|---|---|
| 1 | Открыть URL http://localhost:8000/?balance=30000&reserved=0  | Загружается страница F-Bank. |
| 2 | Кликнуть на карточку «Рубли» | Открывается форма «Перевод на карту» с полем номера карты и видимой 16-символьной маской. |
| 3 | Попробовать ввести 17 цифр подряд (например, 12345678901234567) | После ввода 16-й цифры поле блокирует ввод 17-й. Значение поля содержит ровно 16 цифр. |

## TS-04
**Успешный перевод валидной суммы**

**Приоритет**: 🔴 Critical

**Описание**: Проверка основного сценария перевода: пользователь вводит валидный номер карты и сумму, не превышающую доступную (с учетом комиссии), и подтверждает перевод.

**Предусловия**:
- Сервис F-Bank запущен на http://localhost:8000.
- Используется браузер Google Chrome (последняя версия).

| # | Шаг | Ожидаемый результат |
|---|---|---|
| 1 | Открыть URL http://localhost:8000/?balance=30000&reserved=20001  | Загружается страница F-Bank. На рублевой карточке: «На счету: 30'000 руб., Резерв: 20'001 руб.» |
| 2 | Кликнуть на карточку «Рубли» | Открывается форма «Перевод на карту». |
| 3 | В поле «Номер карты» ввести 16 цифр: 1234567890123456 | Появляется поле «Сумма перевода». |
| 4 | В поле «Сумма перевода» ввести значение 1000 | Отображается «Комиссия: 100 руб.». Сумма с комиссией (1000 + 100 = 1100) не превышает доступную (9999). Появляется кнопка «Перевести». |
| 5 | Нажать кнопку «Перевести» | Появляется системный диалог браузера с заголовком «Подтвердите действие на localhost:8000» и сообщением «Перевод 1000 руб. на карту 1234567890123456 принят банком!». |

## TS-05
**Сообщение об ошибке при превышении доступной суммы**

**Приоритет**: 🟡 Major

**Описание**: Проверка отображения корректного сообщения, когда сумма перевода с учетом комиссии превышает доступную сумму на счете.

**Предусловия**:
- Сервис F-Bank запущен на http://localhost:8000.
- Используется браузер Google Chrome (последняя версия).

| # | Шаг | Ожидаемый результат |
|---|---|---|
| 1 | Открыть URL http://localhost:8000/?balance=30000&reserved=20001  | Загружается страница F-Bank. Доступная сумма = 9999 руб. |
| 2 | Кликнуть на карточку «Рубли» | Открывается форма «Перевод на карту». |
| 3 | В поле «Номер карты» ввести 16 цифр: 1234567890123456 | Появляется поле «Сумма перевода». |
| 4 | В поле «Сумма перевода» ввести значение 10000 | Отображается «Комиссия: 1000 руб.». Сумма с комиссией (10000 + 1000 = 11000) превышает доступную (9999). Под полем суммы отображается красный текст «Недостаточно средств на счете». Кнопка «Перевести» отсутствует. |
